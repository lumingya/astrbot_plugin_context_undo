# Context Undo

`astrbot_plugin_context_undo` 是一个给 AstrBot 使用的上下文撤回插件。

它会在每次 AI 成功回复后，为当前对话保存一个可回退的“编号快照”。当你需要撤回时，不是简单地裁掉最近几条记录，而是直接把对话恢复到指定编号之前的状态，让撤回结果更稳定、更可控。

## 这个插件能做什么

- 为每次成功的 AI 回复生成递增编号，例如 `#1`、`#2`、`#3`
- 按编号回滚当前对话上下文
- 支持查看当前对话有哪些编号可以撤回
- 支持一次撤回最近多条上下文
- 在切换对话或重启 AstrBot 后，保留各自对话的撤回记录
- 回滚时同时恢复相关 LTM 状态，尽量减少“看起来撤回了，但模型还记得”的情况

## 适合什么场景

- AI 刚刚答偏了，想撤回最近一次回复带来的上下文影响
- 想回到某一轮对话之前，重新继续聊
- 群聊或长期对话里，希望上下文撤回更可靠
- 不想因为普通消息里出现“撤回”“回滚”两个字就被误触发

## 使用方法

### 自然语言命令

- `撤回`
- `回滚`
- `撤回列表`
- `回滚列表`
- `撤回 #3`
- `撤回 最近 2`

### 斜杠命令

- `/undoctx`
- `/undoctx list`
- `/undoctx 3`
- `/undoctx last 2`

## 命令说明

- `撤回` / `回滚`
  回滚最新一条已编号上下文。

- `撤回列表`
  查看当前对话最近可撤回的编号。

- `撤回 #3`
  回滚到 `#3` 之前，同时移除 `#3` 以及之后的所有已编号上下文。

- `撤回 最近 2`
  一次回滚最近 2 条已编号上下文。

## 使用示例

1. 你和 AI 连续对话了几轮。
2. 插件已经为这些回复依次记录为 `#1`、`#2`、`#3`……
3. 如果你觉得从 `#3` 开始的内容都不想保留，可以发送 `撤回 #3`。
4. 插件会把当前对话恢复到 `#3` 之前的状态。

如果你只是想先看看能撤回到哪里，可以先发送 `撤回列表`。

## 插件特点

- 撤回目标明确：按编号回退，而不是猜测应该删掉多少历史记录
- 对话隔离：不同对话分别保存，不容易串
- 重启可续：AstrBot 重启后，原对话的编号记录仍可继续使用
- 误触更少：自然语言触发只接受明确格式，不会因为普通聊天里提到相关词语就执行撤回

## 注意事项

- 仅管理员可以使用撤回相关命令
- 每个对话最多保留最近 `48` 条已编号上下文
- 当当前会话被 `reset`、`del`、`delete` 清空时，对应撤回记录也会一起清空
- 如果历史被清空，该对话的编号会重新从 `#1` 开始

## 一句话总结

这是一个让 AstrBot “上下文撤回”更稳定的插件。你可以把它理解成：为每次 AI 回复建立检查点，然后按编号回到你想回去的位置。
